# Career Portfolio

This context defines the language for Andrew Valenzuela's professional portfolio. It keeps the work presentation focused on hiring clarity, project evidence, and honest design proof.

## Language

**Career Portfolio**:
A professional site that presents Andrew's role, selected work, strengths, and contact path for hiring or collaboration.
_Avoid_: Personal website, portfolio app

**Project**:
A demonstrable work sample with a clear purpose, visible result, and reviewable implementation or live experience.
_Avoid_: Activity, output

**Featured Project**:
A Project selected for the first impression because it best proves Andrew's full-stack capability, applied AI experience, or product-building range.
_Avoid_: Favorite project, latest project

**Design Work**:
A Figma prototype, interface study, or visual design artifact that shows product thinking and UI/UX judgment.
_Avoid_: Fake project, placeholder project

**Open to Opportunities**:
The portfolio status that signals Andrew is available for internships, junior developer roles, collaborations, or relevant project work.
_Avoid_: Available for projects

## Example Dialogue

Developer: "Should AGRI-VISION be a Featured Project?"

Domain expert: "Yes. It is a Project with a live experience and source code, and it proves full-stack delivery with applied AI."

Developer: "Should a future Figma idea appear as Design Work?"

Domain expert: "Only if it is presented honestly as upcoming work. Finished Design Work should show a real prototype or study."
